## SatCLIP: Embedding Extraction Tutorial

### Objective

The goal of this experiment is to generate geospatial embeddings using **SatCLIP**, a foundation model that learns geographic representations from latitude and longitude coordinates.

Unlike models such as AlphaEarth, which require multiple Earth observation data sources, SatCLIP only requires geographic coordinates as input:

$$
(\text{longitude}, \text{latitude})
\rightarrow
\text{SatCLIP}
\rightarrow
\text{Embedding}
$$

### Embedding Extraction Code

```python
from pathlib import Path
import sys

import pandas as pd
import torch
from huggingface_hub import hf_hub_download

sys.path.append(str(Path("satclip").resolve()))

from load import get_satclip


device = "cuda" if torch.cuda.is_available() else "cpu"

locations = [
    ("ITAM_CDMX", -99.1840, 19.3460),
    ("Paris", 2.3522, 48.8566),
    ("Tokyo", 139.6917, 35.6895),
    ("New_York", -74.0060, 40.7128),
    ("Monterrey", -100.3161, 25.6866),
]

coords = torch.tensor(
    [[lon, lat] for _, lon, lat in locations],
    dtype=torch.double,
)

checkpoint = hf_hub_download(
    "microsoft/SatCLIP-ViT16-L40",
    "satclip-vit16-l40.ckpt",
)

model = get_satclip(checkpoint, device=device)
model.eval()

with torch.no_grad():
    embeddings = model(coords.to(device)).detach().cpu()

print("Embeddings shape:", embeddings.shape)
```

### Interpretation

The individual dimensions of the embedding do not have a direct physical interpretation.

For example, it is not possible to state that:

```text
Dimension 17 = vegetation
Dimension 42 = temperature
Dimension 89 = population density
```

Instead, the embedding should be viewed as a latent representation learned during contrastive pretraining.

The value of the embedding comes from the relationships between vectors rather than the meaning of any single dimension.

---

### Key Takeaway

This experiment demonstrates that SatCLIP can transform geographic coordinates into dense numerical representations without requiring satellite imagery during inference.

The resulting embeddings provide a compact representation of geographic context that can be used as input for machine learning models and compared against embeddings generated by other Earth foundation models such as AlphaEarth, Clay, Prithvi, GeoCLIP, and MOSAIKS.
